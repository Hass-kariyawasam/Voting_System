# Parliamentary_voting_system
Parliamentary Voting System
This project is a C language simulation of the Sri Lankan Parliamentary Voting System. It models the complete electoral workflow, from voter registration to final result declaration, based on Sri Lanka’s proportional representation system with preferential voting.

Features
User Roles: Admin, Returning Officer (RO), Presiding Officer (PO), Voter, Candidate, Party Agent
Voter Registration: NIC-based validation, age and citizenship verification, duplicate prevention
Candidate & Party Registration: Nomination approval by Returning Officers, unique candidate codes, party registry
Polling Process: Secure electronic ballot casting with up to 3 preferential votes per voter

Vote Counting:
District-level proportional seat allocation using the Largest Remainder (Hare Quota) method
5% threshold enforcement
Bonus seat allocation to the highest-vote party per district
29 National List seats
Result Management: Automated calculation, secure result declaration, and report generation
Data Handling: Stores election data in structured text files (no external database required)

System Interfaces
Admin Panel: District & party registry, system configuration, final result declaration
RO Interface: Nomination approval, candidate management, district-level vote counting
PO Interface: Polling station operations, voter authentication, report generation
Voter Interface: Secure login, preferential vote casting, vote receipt generation

Technical Details
Language: C
Storage: Text file-based database (e.g., voters.txt, candidates.txt, parties.txt, districts.txt)
Design: Menu-driven interface with role-based access control
Security: Password-protected user accounts, NIC validation, data integrity checks
